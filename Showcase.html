  • The "All Players To String" invention allows users to get all players in the instance's display names in a list. The display names are ordered the same as the "Get All Players" chip and is formatted for almost any use.

  • The "Consumable Ordering V2" invention, which is currently unreleased, is a reworked version of the "Consumable Ordering System" invention. This new rework provides documentation for the invention, a rework of the logic inside, less CPU and Network Heat usage, and extra customizability.

  • The "String Writer" invention allows users to simulate a subtitle being typed out infront of them. Useful for welcome messages, server notifications, and more.

  • The "Lighting V2" invention simulates the Cv1 State Machine, typically used to cycle through different lighting states. This Cv2 recreation allows users to save ink on state machines and avoid using the currently broken Cv2 lighting chips.

  • The "RankSystemV2.3" invention gives military roleplay games the ability to neatly and easily manage their in-game ranks. Equiped with the US Army's real world ranks and the ability to create a fully custom rank and regiment, it's quite versatile. Currently undergoing a redesign dubbed "RankSystemV3", rooms that center around military simulation will have an easy upgrade from an already amazing system, bringing the genre to a whole new standard.

  • The "XP Level Up Curve" invention delves more into the math side of Cv2 and simulates a graph to determine a player's XP requirement to level up. Using the somewhat simple equation (x^3 * a)+(x^2 * b)+(x * c), with "x" being the player's next level and "a", "b", and "c" being the three inputs, it lets the player have an increasingly difficult leveling system that goes on [in theory] forever!
